# Governance

This repository uses lightweight project governance for an individual technical contribution.

## Editors and Maintainers

Specification editors maintain the Internet-Draft text. Repository maintainers triage issues, review pull requests, maintain build automation, and ensure generated artifacts remain reproducible.

Richard M. Infantado is the confirmed primary editor. Additional editors require complete author metadata and explicit agreement before being listed in the draft metadata.

## Public Review

Architecture changes should be discussed in public issues or pull requests. Substantive changes should be accompanied by written decision records when they affect interoperability, conformance, security, privacy, data portability, or normative behavior.

## Consensus-Oriented Process

The project seeks rough consensus among active reviewers, but it does not claim IETF working-group adoption or IETF governance. Unresolved objections should be recorded rather than silently ignored.

## Draft Releases

Draft releases should be versioned by Internet-Draft revision and tagged in the repository. Generated RFCXML, text, and HTML artifacts should correspond to the editable source for that revision.

## Normative Changes

Normative behavior must not change silently. Pull requests that add, remove, or modify normative language should identify conformance consequences and should include relevant security and privacy analysis.
